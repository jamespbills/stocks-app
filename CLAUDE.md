Desktop stock monitoring app. Electron + React + TypeScript + Tailwind. Direct MySQL (`new_stocks_db`) via mysql2 in main process. Single user. Dark mode only.

**Think Before Coding.** State assumptions explicitly before implementing. If the request is ambiguous or has more than one reasonable answer, surface the tradeoff and ask — do not pick silently and run. Push back when something looks wrong.

**Surgical Changes.** Touch only what the task requires. Every changed line must trace directly to the request. Match the existing style of the file you're editing. Do NOT refactor adjacent code that wasn't broken, fix formatting in untouched files, or delete pre-existing dead code — mention it instead. Before editing more than 3 files, confirm scope first.

**Goal-Driven Execution.** I give you success criteria, not step-by-step instructions. Loop until they are met. For multi-step tasks, write a plan to `tasks/todo.md` and wait for confirmation before editing files.

## 4. Code conventions

- **TypeScript:** strict mode, no `any`, `interface` for object shapes, `type` for unions/aliases.
- **Naming:** `camelCase` variables/functions, `PascalCase` components/interfaces, `UPPER_SNAKE_CASE` constants.
- **React:** functional components only, hooks for all state. No class components.
- **Tailwind:** utility classes only, no inline `style={}`, no custom CSS unless Tailwind cannot do it.
- **IPC channels:** `namespace:action` pattern — e.g. `db:query`, `scripts:launch`, `fs:readFile`. Consistent in both `main.ts` and `preload.ts`.
- **SQL:** parameterised queries only — `pool.execute(sql, [params])`. Never string concatenation. `lower_snake_case` identifiers.
- **Commits:** `feat:`, `fix:`, `refactor:`, `chore:`. No AI signatures, no co-author tags, no "Generated by Claude".

## 5. Verification loop

Run after **every** change before saying it's done:

```bash
npm run typecheck && npm run lint
```

If either fails, fix it before proceeding. Do not leave type errors or lint warnings.

**UI changes:** open `npm run dev`, navigate to the affected screen, confirm it renders and responds correctly.

**DB changes:** test the query in the dev console via `window.electronAPI.db.query(sql)`. Confirm the row shape matches what the component expects.

**New IPC channel:** confirm the channel name is identical in `ipcMain.handle`, `preload.ts`, and the renderer call site.

**New module:** navigate to it in dev mode. Confirm lazy-loading works (no import errors, placeholder renders).

If verification fails after 3 attempts, stop. Report: what you tried, the failing output, and your hypothesis. Do not guess a fourth time.

## 6. Hard guardrails

- **Never query MySQL from the renderer.** All DB access via `window.electronAPI.db.query()` (IPC → main). Zero exceptions.
- **Never write to financial tables:** `fact_reports`, `fact_metrics`, `stock_archive_flat`, `dim_companies`, `dim_sectors`, `dim_play_definitions`, or any `dim_*` financial dim. App writes to `app_*` tables only.
- **Never hardcode `"play"` or `"play_2"`.** Play strategies are data-driven from `dim_play_definitions`. If you're about to write either string as a hard-coded identifier — stop, query the table instead.
- **Never hardcode file paths.** All external paths come from `config.local.json` → `AppConfig`. Never fall back to a hardcoded string.
- **Never commit `config.local.json`.** It is git-ignored. Never put real credentials in `config.example.json`.
- **No SQLite.** All app state in `new_stocks_db` (`app_*` tables). No exceptions.
- **No new backend service.** No Express, no Flask, no localhost API. Subprocess calls go via IPC only: `scripts:launch` for user-defined scripts from config, `scripts:launchBuiltin(name, args?)` for app-local scripts in `stock-app/scripts/` (resolves path from `app.getAppPath()`, injects DB credentials + `FINNHUB_API_KEY` + `FMP_API_KEY` as env vars automatically; optional `args[]` are passed positionally to the script).
- **No ORM.** Direct mysql2 only. Do not install Prisma, Drizzle, Sequelize, or TypeORM.
- **No light mode.** Dark only. Do not add a theme toggle.
- **Never modify files in `Database Project\`.** Read-only reference.
- **No documentation files** (README, CHANGELOG, etc.) unless explicitly asked.

## 7. Module system

Each module in `src/modules/<module-id>/` with `index.tsx` as default export. Register in `src/module-registry.ts`. That's it — nothing else.

Modules are lazy-loaded via `React.lazy`. Never import a module directly in `app.tsx` or `router.tsx`.

**Before reinventing, reach for these:**
- `src/lib/format.ts` — `formatDate(v, 'long'|'short'|'iso')`, `formatPercent(v, { signed, digits })`. Handles the mysql2 Date-vs-string runtime quirk.
- `src/components/QueryState.tsx` — wraps `useIpcQuery` results; pass the whole query, render with a render prop. Use this for any module's primary fetch.
- `src/components/MutedLabel.tsx` — small uppercase muted label. `mono`/`size`/`color`/`as` props. Use instead of inlining the `fontSize/textTransform/letterSpacing` style.
- `src/components/Popover.tsx` — portal-based popover with viewport-aware positioning. Pass `anchorEl: HTMLElement | null` and `onDismiss`; the popover renders into `document.body`, positions itself fixed relative to the anchor, flips to the anchor's left when right placement would clip the viewport, and clamps vertically to stay on-screen. Use for any floating UI anchored to an element that may sit inside an `overflow: hidden` container.
- `src/hooks/useDebouncedSave.ts` — autosave with `flush()` on blur and a "saved Ns ago" tick. Use for any notes/textarea field that persists to MySQL.

## 8. Shell layout (locked — do not change without explicit instruction)

Full rationale and column/interaction details: `@docs/decisions/wireframe-decisions.md`.

**Window chrome:** Frameless Electron window (`titleBarStyle: 'hidden'`). Single 48px header bar. The bar is `-webkit-app-region: drag` across its full width, with `-webkit-app-region: no-drag` on all interactive controls. Platform-specific controls:
- **Windows:** Custom min/max/close buttons rendered in the top-right corner of the bar. Wire to `BrowserWindow.minimize()`, `.maximize()`, `.close()` via IPC.
- **macOS:** `trafficLightPosition` positions system traffic lights within the bar on the left. No custom buttons needed.

**Sidebar:** Fixed 200px, not collapsible in v1. Two groups:
- *Data:* Watching Dashboard · Report Calendar · Sector Signals
- *Tools:* Reviews · Scripts · TA Charts
- Settings item pinned to the bottom, outside the groups
- Active item background: `var(--color-interactive-active)`. Active text: `var(--color-text-primary)`. Inactive text: `var(--color-interactive-text-inactive)`.
- Sector Signals item: shows a new-item count badge (e.g. `14`) when count > 0; shows keyboard hint (`⌘3` / `Ctrl+3`) when count is 0.

**Watching Dashboard — row click:** Opens a slide-over detail panel fixed at 420px from the right. Table narrows but stays visible. Arrow keys navigate rows while the panel stays open. See `wireframe-decisions.md` for column drop rules, keyboard spec, and close behaviour.

**Report Calendar — overflow:** Days with 3+ entries show 2 inline + a `+N more` pill. Clicking the pill opens a day popover listing all entries.

**Report Calendar — upcoming panel:** Right-side panel (overdue + upcoming lists) is toggleable. Visible by default. Toggle button in module header (`⌘\` shortcut). Preference persists per session.

**Design tokens:** All colour, typography, and spacing values live in `src/styles/tokens.css` as CSS custom properties, matching `@docs/decisions/design-tokens.md` exactly. Tailwind references them via arbitrary value syntax (`text-[var(--color-text-primary)]`) or `@apply` with extended config. Never hardcode a colour value anywhere — always reference a token.

## 9. Known mistakes (grows over time — propose additions after corrections)

> Empty at build start. Each entry follows the pattern: *what Claude did wrong → the rule.*

- *(When Claude tries to query MySQL directly in a React component, remind it: all DB access is IPC only — see §6.)*
- *(When Claude adds a light mode toggle "just in case", remind it: dark only — see §6.)*
- *(When Claude creates a README or guide file unprompted, remind it: no docs unless asked — see §6.)*
- *(When Claude hardcodes a colour value instead of a CSS custom property, remind it: all colours are tokens — see §8.)*
- *(When Claude imports Electron APIs — `webFrame`, `shell`, `nativeTheme`, etc. — directly in renderer/React code, remind it: the renderer is sandboxed and cannot import from `electron`. All Electron APIs must go through `preload.ts` → `contextBridge` → `window.electronAPI`.)*
- *(When Claude implements window controls without platform branching, remind it: Windows = custom min/max/close top-right, macOS = system traffic lights — see §8.)*
- *(When Claude sorts or compares a mysql2 DATE column, remind it: the value is a JS `Date` object at runtime, not a string. Use `instanceof Date` / `.getTime()` — never `String(v).localeCompare()`.)*
- *(When Claude reads a mysql2 DECIMAL column, remind it: the value is a **string** at runtime (e.g. `"14.0000"`), not a number. Strict equality (`=== 13`) will never match. Always parse at the data-transform boundary: `Math.round(Number(row.field))`. Applies to any column typed DECIMAL in MySQL — play scores, percentages, ratios, etc.)*
- *(When Claude calls `.toISOString()` on a mysql2 Date value for display or storage, remind it: `.toISOString()` is UTC-only and will be off by one day in BST. Use `formatDate(value, 'iso')` from `src/lib/format.ts` — see §7.)*
- *(When Claude uses `ticker` as a React key or selection identity in a table, remind it: ticker is not unique — `view_watching` can return multiple rows per ticker. Use a composite `rowKey` — see `tasks/lessons.md`.)*
- *(When calling `scripts:launch`, the first arg is the script file path — not `'python'`. The handler hardcodes `python` as the executable: `spawn('python', [scriptPath, ...args])`. For app-local scripts in `stock-app/scripts/`, use `scripts:launchBuiltin(name, args?)` instead — it resolves the path and injects DB creds + API keys as env vars.)*
- *(When Claude reads `ref.current` directly in a component's render body, remind it: `react-hooks/refs-during-render` blocks this. Use a ref **callback** that updates `useState` (`<div ref={setEl} />`), or for per-key collections keep a `useRef<Map>` for assignment and sync the active element into state via `useLayoutEffect`. See `tasks/lessons.md`.)*
- *(When Claude places a popover/tooltip with `position: absolute` inside a `DayCell`-style cell or any container with `overflow: hidden`, remind it: the popover will be silently clipped to nothing. Use `src/components/Popover.tsx` — it portals to body and positions itself viewport-aware.)*
- *(When Claude uses a bare `window.addEventListener('click', dismiss)` for outside-click handling, remind it: React's `e.stopPropagation()` does NOT stop native bubbling to `window`. The window handler must check `popoverEl.contains(e.target)` before dismissing, or every click inside the popover will also close it.)*
