# Lessons

- **mysql2 DATE columns return JS `Date` objects at runtime**, not strings â€” even though TypeScript types them as `string | null`. Any sort, comparison, or string operation on a date field must handle `instanceof Date` (e.g. coerce via `.getTime()`). Do not rely on `String(value).localeCompare()` for date sorting.

- **`ticker` is not a unique row identifier.** `view_watching` can return multiple rows for the same ticker (one ticker, multiple active reports). Never use `ticker` alone as a React `key`, selection state, or `findIndex` target. Always derive a composite `rowKey` â€” e.g. `` `${ticker}|${filing_identifier ?? date_released ?? ...}` ``.

- **`useState(prop)` only initialises on mount â€” don't mount with a placeholder, then update the prop.** If a component's local state is seeded from a prop (`useState(detail.comboNote)`), and you mount the component with an empty/placeholder version of that prop before async data arrives, the state will be stale for the component's lifetime (even after the prop updates in the parent). Fix: delay mounting the component until real data is available. A loading placeholder (rendered when `selectedCombo && !comboDetail`) is cleaner and safer than an immediately-mounted shell.

- **Never call `.toISOString()` on a mysql2 Date value for display or intermediate storage.** `.toISOString()` converts to UTC, which shifts the date one day back in BST (UTC+1). Use `formatDate(value, 'iso')` from `src/lib/format.ts` instead â€” it uses local date getters (`getFullYear`, `getMonth`, `getDate`) and is correct in all timezones.

- **react-hooks/set-state-in-effect (ESLint plugin v7.1.1) blocks synchronous `setState` calls in `useEffect` bodies.** Restructure fetches to use `.then()` callbacks, or ensure all state updates are inside the async callback, not synchronously before it. `set-state-in-effect` fires on any setState call that isn't inside an async callback or event handler inside the effect.

- **react-hooks/refs-during-render (same plugin) blocks reading `ref.current` during render.** It only allows ref reads inside event handlers, effects, or layout effects. When a child component needs a stable DOM element from a parent (e.g. anchoring a popover to its trigger), the cleanest fix is a ref **callback** that updates `useState`: `const [el, setEl] = useState<HTMLDivElement | null>(null); <div ref={setEl} />`. For dynamic per-key collections (e.g. one element per ticker on the calendar), keep a `useRef<Map>` for assignment, then sync the *active* element into state via a `useLayoutEffect` keyed on the active identifier. Never assign `ref.current = â€¦` inside the function body.

- **Popovers whose anchor sits inside an `overflow: hidden` container must portal to `document.body`.** Absolute positioning relative to the anchor is silently clipped to nothing. Use `createPortal`, `position: fixed`, and compute `top`/`left` from `anchorEl.getBoundingClientRect()` in a `useLayoutEffect`. Always flip to the anchor's left side when the right placement would clip the viewport, and clamp the top so the popover stays on-screen near the bottom edge. Reposition on `scroll` (capture phase) and `resize`. See `src/components/Popover.tsx`.

- **Outside-click dismiss via a global `window` click listener must check `contains(e.target)`.** Bare `onClick={(e) => e.stopPropagation()}` on the popover root does NOT stop native bubbling to `window` in React 17+ (events are delegated at the root container, not document). Without the containment check, every click inside the popover would also dismiss it.

- **For Disregard pre-fill on the Report Calendar, derive YEAR from the year part of the report's relevant date and FILING from `view_earnings_calendar.next_expected_filing`** â€” never from `r_financial_year` or `r_filing_identifier`, which reflect the company's currently-tagged FY and the *last* filed report respectively. The Disregard form is editable so this is a best-guess pre-fill, but defaulting to the next-upcoming report's year/filing is what the user expects.
